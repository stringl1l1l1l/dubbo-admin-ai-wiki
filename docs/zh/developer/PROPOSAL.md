# Proposal 模板

本模板用于记录“值得被长期保留”的变更设计。适用对象通常不是小修小补，而是：

- 架构调整
- 核心接口变更
- 组件边界调整
- 数据模型演进
- 上线和迁移策略

一份好的 Proposal，目标不是把内容写长，而是让后来者能快速回答三件事：

1. 当时为什么要做这件事
2. 最终为什么选了这个方案
3. 这个方案带来了哪些新约束和后续任务

## 1. 背景

说明当前问题、业务上下文、触发原因和已有痛点。

建议回答：

- 问题出现在哪里
- 为什么现在必须处理
- 如果不处理，会带来什么后果

## 6. 可观测性

这部分不要省略。任何进入主链路的方案都应回答：

- 需要新增哪些日志
- 需要哪些指标
- 是否需要 trace 关联
- 如何判断方案运行正常
