# Proposal Template

A useful proposal is not supposed to be long. It should let later readers answer four questions quickly:

1. Why this change was needed
2. Why this option was chosen
3. What new constraints and follow-up work it introduces
4. How the feature should be implemented

## 1. Background

Describe the current problem, business context, trigger, and existing pain points.

Suggested prompts:

- Where does the problem appear
- Why must it be handled now
- What happens if it is left unresolved
